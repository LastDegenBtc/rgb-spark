# CLAUDE.md

Guidance for Claude Code working in this repo.

## What this repo is

`rgb-spark` is the brand-neutral, self-custody **RGB-on-Spark**
reference wallet. It's the sibling of `spark-spike` (custodial
PPwallet at `wallet.pprgb.app`); the two are intentionally distinct
products with opposite trust models, not stages of one migration.

The core construction is **Spark-UTK**, specified in [`SPARK-UTK.md`](./SPARK-UTK.md).
The Rust implementation lives in vendored forks under `forks/`
(`bp-core` and `rgb-consensus`, both on the 0.11.1-alpha.2 line for
Bitmask compatibility) plus our wrapper crate `rgb-spark-core` that
compiles the primitives to WASM.

## Common commands

Build the WASM bundle and vendor it into the frontend:

```bash
bash scripts/build-spark-core.sh           # release build (default)
PROFILE=dev bash scripts/build-spark-core.sh
```

Frontend (`frontend/`):

```bash
cd frontend
npm install
npm run dev          # Vite dev server — ALWAYS SAFE, never touches dist/
npm run build:check  # sandboxed compile → dist-check/ (gitignored)
npm run lint
```

**`npm run build` is guarded** — refuses to run unless `CONFIRM_PROD_BUILD=1`
is set. The guard is installed before there's even a prod wired up
because the same mistake that hit `spark-spike` on 2026-05-11 will
happen again the day this repo *does* have a prod. See
`feedback_prod_build_guard.md` in the agent memory.

Rust tests (under `forks/rgb-spark-core/`):

```bash
cd forks/rgb-spark-core
cargo test                                  # native — 4 deterministic vectors
cargo build --target wasm32-unknown-unknown # WASM compile-only sanity
```

## Architecture pointers

- `forks/bp-core/dbc/src/keytweak/mod.rs` — Spark-UTK keytweak primitives
  (`derive_u_tweaked`, `derive_verifying_key`, `derive_output_xonly`),
  `SparkUtkProof` strict-encoded type, all unit-tested with
  deterministic vectors pinned in `scoping/03-test-vectors.md`.
- `forks/rgb-consensus/src/validation/commitments.rs` — `DbcProof::SparkUtk`
  variant (tag `0x03`).
- `forks/rgb-spark-core/src/lib.rs` — `wasm-bindgen` surface:
  `deriveUTweaked` / `deriveVerifyingKey` / `deriveOutputXonly` /
  `SparkUtkProofJs` with `encode/decode` round-trip matching the
  rgb-consensus on-the-wire bytes.
- `frontend/src/lib/spark-core/` — vendored output of `wasm-pack`,
  consumed by the wallet. Regenerated by `scripts/build-spark-core.sh`.

## Repo discipline

- Don't propose carriers, leaf-tweaks, or custodial fallbacks — those
  are the `spark-spike` product, not this one.
- Don't pull `bitmask-core` or any other RGB-on-L1 transport into the
  browser. Spark replaces the transport layer; that's the whole point.
- Don't add the `pp` brand to anything; this repo is positioned as
  an industry-neutral reference implementation, potentially the basis
  for an RGB-WG proposal.
- The vendored `forks/bp-core/` and `forks/rgb-consensus/` are pinned
  to 0.11.1-alpha.2 for Bitmask wire-compatibility. Don't rebase to
  0.12 — that's a different RGB camp.
